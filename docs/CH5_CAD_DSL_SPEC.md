## 1. Problem — 直接生成 Python 的脆弱性

Phase IV 讓 LoRA-B 直接輸出 build123d Python；失敗模式分三類：

| 類型 | 範例 | 後果 |
|------|------|------|
| 語法錯誤 | 缺括號、縮排錯 | `SyntaxError`，pipeline 中止 |
| Import 錯誤 | `from build123d import Boxs` | `ImportError`，pipeline 中止 |
| 執行期崩潰 | 負值壁厚、零體積 | `OCCT Error`，STL 不生成 |

三類錯誤在 `export_stl_high_density` 後才被抓到（late failure）。
JSON 有 schema，可在執行前 100% 靜態查錯——這是 CH5 的核心動機。

## 2. Proposed DSL — JSON Sketch-Extrude

### 2.1 頂層結構

```json
{
  "$schema": "cadhllm/sketch_extrude_v1",
  "version": 1,
  "label": "shell_bottom_120x90x30",
  "operations": [],
  "material": "PLA"
}
```

### 2.2 操作節點（`op` 欄位決定種類）

| op | 必填欄位 | 說明 |
|----|---------|------|
| `sketch_rect` | `plane`, `w`, `h` | 矩形 sketch（`plane`: `XY\|XZ\|YZ\|offset_z:<mm>`）|
| `sketch_circle` | `plane`, `r` | 圓形 sketch |
| `extrude` | `amount` | 線性擠出（`amount != 0`，`ref` 引用前一 label）|
| `shell_op` | `thickness`, `open_faces` | 殼化（`thickness ∈ [1.5, 4.0]`）|
| `fillet` | `radius`, `target` | 圓角（`radius ∈ (0, 10]`，`target`: `all_edges\|top_edges\|…`）|
| `chamfer` | `size`, `target` | 倒角（`size ∈ (0, 5]`）|
| `cut` | `tool` | 布林差集（`tool`: 前一個 solid label）|
| `union` | `parts` | 布林聯集（`parts`: label 陣列）|
| `translate` | `v` | 位移（`v: [dx, dy, dz]`）|
| `rotate` | `axis`, `angle_deg` | 旋轉（`axis: X\|Y\|Z`）|

每個節點可選 `"label": "<str>"` 供後續節點 `ref`/`tool`/`parts` 引用。

## 3. Example — 殼體底板

### 現行 Python（脆弱）

```python
from build123d import *
with BuildPart() as box:
    Box(120, 90, 30)
    offset(amount=-2.0, openings=box.faces().sort_by(Axis.Z)[-1])
    fillet(box.edges().filter_by(Axis.Z), radius=2.0)  # LLM 可能誤寫 "Z"
```

### DSL JSON（可提前驗證）

```json
{
  "$schema": "cadhllm/sketch_extrude_v1", "version": 1,
  "label": "shell_bottom", "material": "PLA",
  "operations": [
    { "op": "sketch_rect", "label": "sk", "plane": "XY", "w": 120, "h": 90 },
    { "op": "extrude",     "label": "box", "ref": "sk", "amount": 30 },
    { "op": "shell_op",    "label": "body", "thickness": 2.0, "open_faces": ["top"] },
    { "op": "fillet",      "label": "flt",  "radius": 2.0, "target": "all_edges" }
  ]
}
```

`w>0`、`thickness∈[1.5,4.0]`、`radius≤10` 在 transpile 前已由 jsonschema 驗畢。

## 4. Transpiler 設計（DSL → build123d Python）

**原則**：純確定性（相同 JSON → 相同 Python）、無 LLM 參與、無 build123d 依賴（可 unit-test）。

### 模組位置

```
lib/cad/dsl/
├── schema.py       # JSON Schema 常數 + validate_dsl() -> (ok, errors)
├── transpiler.py   # compile_dsl_to_python(dsl_dict) -> str
└── __init__.py
```

### 輸出格式

```python
# [auto-generated by cadhllm DSL transpiler v1]
from build123d import *
with BuildPart() as _p:
    with BuildSketch(Plane.XY) as sk:
        Rectangle(120, 90)
    extrude(amount=30)                                      # box
    offset(amount=-2.0, openings=_p.faces().sort_by(Axis.Z)[-1:])  # body
    fillet(_p.edges(), radius=2.0)                          # flt
_result = _p.part
```

transpiler 維護 `_label_map: dict[str, str]` 追蹤 label → Python 變數，
供 `cut`/`union`/`ref` 節點解析引用。

### 錯誤層次

| Layer | 攔截點 | 攔截內容 |
|-------|--------|---------|
| 1 | `validate_dsl()` — JSON parse | 非法 JSON |
| 2 | `validate_dsl()` — jsonschema | 缺必填欄位、值越界 |
| 3 | `transpiler._resolve_ref()` | label 引用不存在 |
| 4 | phase4_handler try/except | build123d 幾何錯誤（last resort）|

目標：Layer 1–3 覆蓋 ≥95% 的 LLM 輸出錯誤。

## 5. Integration — Phase IV 修改點

### bridge 新增欄位

```json
"cad_output": {
  "dsl_output": {},   // 原始 DSL JSON
  "dsl_valid": true,
  "dsl_errors": [],
  "dsl_python": ""    // generated code
}
```

### phase4_handler 路由（flag-based）

```python
USE_DSL = os.getenv("CADHLLM_USE_DSL", "0") == "1"

if USE_DSL and "dsl_output" in bridge.get("lora_b_raw", {}):
    ok, errors = validate_dsl(bridge["lora_b_raw"]["dsl_output"])
    if ok:
        part = _exec_dsl_python(compile_dsl_to_python(...))
    else:
        # DSL 驗證失敗 → fallback，pipeline 不中止（PV3 豁免）
        part = _solver_fallback(...)
else:
    part = _solver_fallback(...)   # 現行路徑不變
```

### LoRA-B 訓練資料

新增 output mode token：
- `<|im_start|>dsl` → 模型輸出 DSL JSON
- `<|im_start|>python` → 現行 Python（向後相容）

由 training config `lora_b_output_mode: "dsl" | "python"` 控制。

## 6. Migration 計畫

| 階段 | `CADHLLM_USE_DSL` | 行為 |
|------|------------------|------|
| Phase 0（現在）| `0` | 全走 solver fallback，DSL 模組不啟用 |
| Phase 1（DSL 落地）| `1` | valid → DSL path；invalid → fallback |
| Phase 2（LoRA-B 訓完）| `1`（default）| DSL 為主，Python mode 保留回退 |

---

## 7. Scope Limitation

| 幾何種類 | DSL | 仍需 Python |
|---------|-----|------------|
| 矩形 / 圓形殼體 + 擠出 | ✅ | — |
| 殼化 / 圓角 / 倒角 | ✅ | — |
| 布林開孔 | ✅ | — |
| Snap-fit 懸臂 | ❌ | `lib/cad/shell`（含應力公式）|
| 螺絲柱 / 線槽 / 通風格柵 | ❌ | `lib/cad/shell/enclosure_vents.py` |
| 機械支架（mount）| ❌ | `lib/cad/mounts.py` |
| 自由曲面 | ❌ | spline，DSL 不支援 |

DSL 只涵蓋「primitives + 擠出 + 布林」；複雜幾何保留 Python。
LLM 不需判斷邊界，routing 由 `has_multi_component_layout()` 統一決策。

## 8. 驗收標準

| 項目 | 方法 | 目標 |
|------|------|------|
| Schema coverage | 10 個合法 DSL unit test | 100% `validate_dsl` pass |
| Early error catch | 10 個含錯 LLM 輸出 | ≥9 被 Layer 1–3 攔截 |
| Round-trip | DSL → Python → exec → STL | 無 OCCT 錯誤 |
| Fallback integrity | DSL invalid 時 | pipeline 不中止，`bridge.dsl_errors` 記錄 |
| Coexistence | `CADHLLM_USE_DSL=0` | 現行所有 phase-test 全過 |
