## 專案背景（5 句話速覽）

StemAiAgent V2 是一套 **AI 輔助 STEM 教育平台**，目標讓國高中學生從「構想→電路→外殼→程式碼」一站完成。平台內建 7 個階段的教學流程（基於 6E 教學模型：Engage → Clarify → Extract → Plan → Schematic → Components-3D/Assembly → Code），每個階段都有 AI 引導。核心 AI 研究為 **CAD-HLLM**（CAD Hierarchical LLM）：用 LoRA 微調大語言模型，讓它能根據學生的元件選擇自動生成 3D 可列印外殼。系統採前後端分離架構，前端為 React（v6/ 目錄），後端為 Python Flask（services/ + lib/）。所有元件規格（尺寸、功率、接腳、重量）統一由 `data/component_datasheet_verified.json` 作為唯一真值來源（SSOT）。

### 5b. SSOT 五欄交叉驗證體系

> **原理**：元件資料庫（SSOT）是所有下游系統的真值來源。如果 SSOT 說 LED 功率 20mA，那麼求解器、渲染器、訓練資料、功率預算、前端顯示必須全部一致。五欄驗證就是分別從五個方向確認這個數字沒走樣。

| 驗證項 | 測試數 | 驗什麼 |
|-------|--------|-------|
| 功率（POWER_MA） | 119 pass + 5 xfail | 45 個元件的功率從 SSOT 到 5 個消費端完全一致 |
| 安裝方式（enclosure_relation） | 39 tests | 17 個檔案中 110 處引用全部指向同一 enum |
| 接腳推導（ConnectorPort） | 229 pass | Arduino 7 port + ESP32 + RPi + Microbit 的接腳推導正確 |
| 散熱（thermal_mw） | 69 tests | SSOT → 元件資料庫 → 組裝求解器三層一致 |
| 重量（weight_g） | 96 tests | SSOT → 元件資料庫雙向覆蓋 + 語義檢查 |

**16 個教學範本的五欄一致性：16/16 通過，0 不一致**
