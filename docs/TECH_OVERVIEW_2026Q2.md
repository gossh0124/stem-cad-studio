## 一、專案是什麼

**StemAiAgent V2**（代號 CADHLLM）是一套**文字轉 CAD 的 STEM 教學平台**。

學生用一句話描述想做的專題（例如「我想做一台自動澆花器」），系統就會自動完成：

1. 理解意圖、選擇元件
2. 產生電路圖、3D 外殼模型
3. 生成可上傳 Arduino 的程式碼
4. 輸出可 3D 列印的 STL 檔案

整個流程遵循 **6E 教學模型**（Engage → Explore → Explain → Engineer → Evaluate → Extend），讓學生在 AI 引導下完成從構想到實體的全過程。

## 三、技術棧

| 層級 | 技術 | 選型理由 |
|------|------|----------|
| **後端框架** | FastAPI (Python) | 原生 async、自動 OpenAPI 文件、SSE 支援 |
| **前端** | React 18 (JSX, no-build) | 輕量、免 webpack 配置、適合教育場景快速迭代 |
| **3D 引擎** | build123d | 純 Python CAD 核心，取代 cadquery，可直接在 Colab 執行 |
| **3D 檢視** | Three.js | 瀏覽器端 WebGL 渲染，支援 STL/GLB 預覽 |
| **電路圖佈局** | ELK (Eclipse Layout Kernel) | 自動層次化佈局，避免手動排版 |
| **LLM** | Llama 3.1 8B Instruct (4-bit) | 開源、可本地部署、LoRA 微調成本低 |
| **微調** | Unsloth + LoRA | r=16, α=32，7 個注意力矩陣，訓練速度快 2-3 倍 |
| **向量資料庫** | LanceDB | 嵌入式、零配置、比 ChromaDB 快 |
| **工作佇列** | SQLite | 輕量、免伺服器、適合單機部署 |
| **即時通訊** | SSE (Server-Sent Events) | 比 WebSocket 簡單、HTTP/1.1 相容、單向串流夠用 |

## 四、核心模組說明

### 4.1 元件資料庫（lib/registry.py — 836 行）

- **50+ 元件規格**，每筆包含：物理尺寸（游標卡尺實測 ±0.5mm）、腳位定義、功率消耗、散熱特性
- **4 種 MCU 板**：Arduino Uno / ESP32 / Micro:bit V2 / Raspberry Pi 4B
- **雙軸標籤系統（U1）**：axis1 = MCU 如何控制（bus/gpio/passive）、axis2 = 元件功能（measure/actuate/display...）
- 所有數據附 datasheet 來源編號（RD3），可追溯

### 4.2 組裝求解器（lib/assembly_solver.py — 732 行）

7 步約束求解流程：
1. **Zoning** — 將元件分區（電源區/信號區/散熱區）
2. **Gravity** — 重心平衡計算
3. **Thermal** — 高發熱元件間距約束
4. **Power Routing** — 電源走線規劃
5. **Signal Routing** — 信號走線（2D A* 演算法）
6. **Collision** — 碰撞檢測
7. **Validation** — 最終驗證（間距/高度/功率）

### 4.3 CAD 引擎（lib/cad/ — 1,540 行）

- **shell.py**：外殼生成（底座 + 上蓋 + 螺絲孔 + 通風孔）
- **pcb_body.py**：PCB 板輪廓（目前以 Arduino 尺寸為主）
- 輸出格式：STL（列印用）、STEP（工程用）、GLB（瀏覽器預覽）

### 4.4 韌體合成（lib/firmware.py — 1,174 行）

- 根據元件組合自動產生 Arduino .ino 程式碼
- **Reaction Rules 機制**：定義感測器→輸出的映射規則，自動生成 `loop()` 邏輯
- 支援 Arduino / ESP32 / Micro:bit / Raspberry Pi

### 4.5 RAG 子系統（lib/rag.py — 668 行）

- Ensemble RAG：結合語義搜尋（向量相似度）+ 關鍵字搜尋
- 用途：使用者輸入模糊描述時，搜尋最匹配的元件規格
- 索引來源：`data/component_datasheet_verified.json`

## 五、前端 UI 架構

### 5.1 主要頁面（8 個視圖）

| 視圖 | 對應 Phase | 功能 |
|------|-----------|------|

- **Demo 模式**：17 個預製範本可直接瀏覽，唯讀不可修改

## 九、已知問題與技術債

### 9.1 當前阻塞項（P0-P1）

| 編號 | 問題 | 影響 |
|------|------|------|
| CV1-CV3 | GLB 多色渲染 + PCB 方向 + 圖層切換 — 未經瀏覽器實測 | 3D 檢視器可能顯示錯誤 |
| DM4 | Pipeline 不支援 `start_from_phase`（中途重跑） | Fork 功能無法使用 |

### 9.2 中等優先（P2）

| 編號 | 問題 | 影響 |
|------|------|------|
| DM2 | 16 範本逐一走查尚未全部完成 | 部分範本可能有未發現的 bug |
| DM3 | 光劍 snap-fit 驗證失敗，無 STL 產出 | 1 個範本不可用 |

### 9.3 測試覆蓋缺口

- **無前端測試**：缺少 Jest / Playwright 等自動化 UI 測試
- **Phase Handler 單元測試不足**：僅有 RAG + 工作佇列的 smoke test
- **端對端測試**：僅有手動驗證腳本，無 CI 自動化

### 9.4 明確的 Scope 外項目

以下功能**刻意不做**，非遺漏：

| 類別 | 項目 | 原因 |
|------|------|------|
| 切片器整合 | G-code 生成、支撐預測 | 交由 Cura/PrusaSlicer 處理 |
| 進階模擬 | FEM 應力分析、跌落測試 | snap-fit 解析已足夠教學需求 |
| 進階殼體 | 多殼分件、戶外防水、多軸機構 | 等 LoRA-B 訓練完成後再評估 |
| PCB 精度 | ESP32 實物校正、Tier 2 模組 | 需採購實物 + 卡尺量測 |

## 十、架構設計亮點

1. **Bridge Pattern**：單一 JSON 物件貫穿 7 個階段，每階段只新增欄位不修改前者，保證資料一致性
2. **SSOT 元件庫**：所有元件規格只存一處（registry.py），前端/後端/訓練共用同一份
3. **Reaction Rules 韌體合成**：宣告式映射（感測器→動作），自動生成可編譯的 Arduino 程式碼
4. **6E 教學整合**：不只是工具，而是把 STEM 教學法嵌入 UI 流程，每個階段對應一個學習環節
